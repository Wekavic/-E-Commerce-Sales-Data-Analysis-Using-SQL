# -E-Commerce-Sales-Data-Analysis-Using-SQL



Project Overview

This project delves into an e-commerce dataset utilizing SQL to extract valuable business insights concerning customer behavior, product performance, and revenue trends. The analysis showcases practical SQL proficiencies, including filtering, aggregation, joins, subqueries, and window functions.

The primary objective of this project is to address pertinent business questions that facilitate informed decision-making in key areas such as:

*   Customer Segmentation: Understanding different customer groups.
*   Revenue Optimization: Strategies to maximize sales and profitability.
*   Product Performance Evaluation: Assessing the success and impact of various products.
*   Regional Sales Analysis: Examining sales patterns across different geographical locations.

🗂 Dataset Description

The dataset comprises four interconnected relational tables, each containing specific information:

| Table Name    | Description                                   | Key Columns                                                              |
| :------------ | :-------------------------------------------- | :----------------------------------------------------------------------- |
| `customers`   | Contains customer demographic information.    | `customer_id`, `first_name`, `last_name`, `email`, `city`, `join_date` |
| `orders`      | Stores details of customer transactions.      | `order_id`, `customer_id`, `order_date`                                  |
| `order_items` | Provides product-level details for each order.| `order_item_id`, `order_id`, `product_id`, `quantity`, `unit_price`    |
| `products`    | Contains the product catalog information.     | `product_id`, `product_name`, `category`, `price`                        |

 🧠 Business Questions Answered

This project addresses several business questions, providing SQL queries and their corresponding insights:

 1. Customers Located in New York

```sql
SELECT *
FROM customers
WHERE city = 'New York';
```

Insight: Identifies customers residing in New York, enabling targeted location-based marketing campaigns.

2. Electronics Products Above $100

```sql
SELECT *
FROM products
WHERE category = 'Electronics'
AND price > 100;
```

Insight: Highlights premium electronics products that contribute significantly to revenue per transaction.

3. Most Recent Customer Orders

```sql
SELECT *
FROM orders
ORDER BY order_date DESC
LIMIT 10;
```

Insight: Offers visibility into the latest purchasing activities and emerging sales trends.

 4. Number of Customers per City

```sql
SELECT city, COUNT(*) AS total_customers
FROM customers
GROUP BY city
ORDER BY total_customers DESC;
```

Insight: Pinpoints cities with the highest customer concentration, aiding in focused expansion strategies.

 5. Revenue Generated by Each Product Category

```sql
SELECT p.category,
SUM(oi.quantity * oi.unit_price) AS total_revenue
FROM order_items oi
JOIN products p
ON oi.product_id = p.product_id
GROUP BY p.category
ORDER BY total_revenue DESC;
```

Insight: Determines the top-performing product categories that are driving revenue growth.

6. Product Categories with More Than 5 Products

```sql
SELECT category, COUNT(*) AS total_products
FROM products
GROUP BY category
HAVING COUNT(*) > 5
ORDER BY total_products DESC;
```

Insight: Identifies categories with extensive product inventories, necessitating structured inventory management.

7. Orders with Customer Names and Dates

```sql
SELECT 
o.order_id,
CONCAT(c.first_name, ' ', c.last_name) AS customer_name,
o.order_date
FROM orders o
INNER JOIN customers c
ON o.customer_id = c.customer_id;
```

Insight: Links customers to their respective orders, fostering a deeper understanding of purchasing behavior.

 8. Total Amount Spent by Each Customer

```sql
SELECT 
c.first_name,
c.last_name,
SUM(oi.quantity * oi.unit_price) AS total_spent
FROM customers c
INNER JOIN orders o
ON c.customer_id = o.customer_id
INNER JOIN order_items oi
ON o.order_id = oi.order_id
GROUP BY c.customer_id, c.first_name, c.last_name
ORDER BY total_spent DESC;
```

Insight: Identifies high-value customers, crucial for developing loyalty programs and retention strategies.

9. Products That Have Never Been Ordered

```sql
SELECT 
p.product_id,
p.product_name
FROM products p
LEFT JOIN order_items oi
ON p.product_id = oi.product_id
WHERE oi.product_id IS NULL;
```

Insight: Detects underperforming products that may require marketing attention or pricing adjustments.

10. Customers Spending Above Average

```sql
SELECT *
FROM (
SELECT 
c.customer_id,
c.first_name,
c.last_name,
SUM(oi.quantity * oi.unit_price) AS total_spent
FROM customers c
JOIN orders o
ON c.customer_id = o.customer_id
JOIN order_items oi
ON o.order_id = oi.order_id
GROUP BY c.customer_id, c.first_name, c.last_name
) customer_totals
WHERE total_spent >
(
SELECT AVG(total_spent)
FROM (
SELECT 
c.customer_id,
SUM(oi.quantity * oi.unit_price) AS total_spent
FROM customers c
JOIN orders o
ON c.customer_id = o.customer_id
JOIN order_items oi
ON o.order_id = oi.order_id
GROUP BY c.customer_id
) avg_table
);
```

Insight: Identifies customers whose spending habits surpass the average customer value.

11. Classifying Orders as Large or Small

```sql
SELECT 
o.order_id,
SUM(oi.quantity) AS total_items,
CASE
WHEN SUM(oi.quantity) > 5 THEN 'Large'
ELSE 'Small'
END AS order_size
FROM orders o
JOIN order_items oi
ON o.order_id = oi.order_id
GROUP BY o.order_id;
```

Insight: Categorizes orders based on their size, facilitating detailed analysis of purchasing patterns.

 12. Top 3 Spending Customers per City

```sql
SELECT *
FROM (
SELECT 
c.city,
c.first_name,
c.last_name,
SUM(oi.quantity * oi.unit_price) AS total_spent,
ROW_NUMBER() OVER (
PARTITION BY c.city
ORDER BY SUM(oi.quantity * oi.unit_price) DESC
) AS rank_num
FROM customers c
JOIN orders o
ON c.customer_id = o.customer_id
JOIN order_items oi
ON o.order_id = oi.order_id
GROUP BY c.customer_id, c.city, c.first_name, c.last_name
) ranked
WHERE rank_num <= 3;
```

Insight: Identifies the top three spending customers in each city, enabling localized marketing strategies.

 📈 Skills Demonstrated

This project effectively demonstrates proficiency in a range of SQL concepts and techniques, including:

*   SELECT filtering: Efficiently retrieving specific data subsets.
*   ORDER BY sorting: Arranging query results in a meaningful sequence.
*   GROUP BY aggregation: Summarizing data based on common attributes.
*   HAVING filtering: Applying conditions to aggregated groups.
*   INNER JOIN: Combining rows from multiple tables based on related columns.
*   LEFT JOIN: Retrieving all rows from the left table and matching rows from the right table.
*   Subqueries: Embedding queries within other SQL queries for complex data retrieval.
*   CASE statements: Implementing conditional logic within SQL queries.
*   Window functions (ROW_NUMBER): Performing calculations across a set of table rows related to the current row.
*   Revenue calculation logic: Deriving total revenue from quantity and unit price.
